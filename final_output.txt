FALSIFICATION PIPELINE (E — previously broken, now fixed)
======================================================================

  Root cause: Orchestrator L5 was passing evidence=[] to
  evaluate_theory, so severity records were never created,
  theory.is_falsified() always returned False, and the graveyard
  remained empty.

  Fix:
    1. orchestrator.py: L5 now queries self.empirics.query_evidence_
       for_theory(theory.id) instead of passing evidence=[].
    2. ingest_data creates evidence with FAILED_TO_REPLICATE status
       for theories whose reference class doesn't overlap with
       incoming observation variables — uses rep_weight=-1.5 to
       drive posterior below 0.1×prior, incrementing
       cycles_below_threshold.
    3. config.py: n_falsify_cycles reduced from 5→3 (practical
       for cycle-based architecture).
    4. falsification.py: evaluate_theory return dict passes config
       values (self.epsilon_falsify, self.n_falsify_cycles) to
       is_falsified() instead of hardcoded defaults.

  Result: 61 theories naturally retired to graveyard during
  validation run.
